Javascript is the synchronous single-threaded programming language. So JavaScript can execute one comment at a time. When we run the JavaScript code the code is stored in the memory heap. From there it's going to call stack where the main code is executed from. In call stack JavaScript code Run line by line in a specific way. Call stack work like LIFO(last in first out) system. But sometimes some functions need extra time to load (example: setTimeout() function). In this situation, JavaScript does not wait for the execution of this code. So JavaScript places the function in the callback queue to finish this process. And after that, it’s go to the next line of code. At this time event loop is continuously running the process . if the call stack is not empty, the event loop waits until it is empty and places the next function from the callback queue to the call stack and execution the code.

Basically, local storage and session storage has many similarities but they have some difference also. The difference is shown below,

localStorage sessionStorage
localStorage is not deleted automatically even we close the browser. sessionStorage is automatically deleted when the browser tab is closed.
localStorage is accessible from any window. sessionStorage is accessible from the browser's same tab only.
localStorage can store data more than sessionStorage. sessionStorage’s memory capacity is lower than localStorage.
localStorage is the same at different tab. sessionStorage is unique per tab.
We can set localStorage with localStorage.setItem("key", "value"). We can set sessionStorage with sessionStorage.setItem("key", "value");.

JavaScript is a single-threaded programming language. That's why it's run the code line by line. JavaScript code can't directly convert into binary code. To convert it as a binary code every browser has a JavaScripts engine. JavaScript engine is simply a computer program that executes JavaScript code and converts it as machine code or binary code. The engine content has two main components, one is the memory heap and another is the call stack. In memory heap, all of our functions and variable are stored . The call stack is where our code is actually executed. The call stack is a data structure that records where is the program we are. In call stack, our code is executed like last in first out.
